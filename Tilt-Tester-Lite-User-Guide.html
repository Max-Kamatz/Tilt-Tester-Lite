Tilt Tester Lite

PTZ Cable Stress Test Tool — User Guide

Automated tilt cycle testing with live Moxa NPort connectivity monitoring

Version 1.0  |  March 2026

1. Overview

Tilt Tester Lite is a Windows desktop application designed to stress-test PTZ camera platform cabling. It continuously cycles the platform tilt between −90° and +90° via Pelco-D over TCP while simultaneously monitoring the connectivity of up to four Moxa NPort serial-to-Ethernet devices through SSH ping sessions on a GENE ARH6 Ubuntu board. All events are logged in real time to a crash-safe CSV file.

Key Features

  • Configurable tilt cycle count (1 – 99,999 cycles)
  • Live per-device connectivity tiles with ping loss and connectivity loss counters
  • Timestamped event log with 10,000-row capacity and auto-scroll
  • Automatic test stop when all four devices have experienced connectivity loss
  • Export log as CSV or Excel (.xlsx)
  • Crash-safe logging — data is preserved even if the application closes unexpectedly

3.2 Device Tiles (left panel)

One tile is displayed per monitored device (10.10.10.2 – 10.10.10.5). Each tile shows:

  • Status badge — colour-coded current state of the device
  • Ping Loss counter — number of individual missed ping replies
  • Connectivity Loss counter — number of times the device reached the connectivity loss threshold
Badge StateMeaning
OKDevice is responding normally to pings
Ping LossOne or more recent ping replies missed (below threshold)
Connectivity Loss5 or more consecutive missed pings — device considered offline
Connectivity RestoredDevice has recovered after a connectivity loss event

3.3 Event Log (right panel)

Displays all test events in real time with timestamp, source, event type, and detail columns. Rows are colour-coded by source. The log holds up to 10,000 rows; older rows are removed automatically. Use the Auto-scroll checkbox to pin the view to the latest entry.

4. Running a Test

4.1 Before Starting

  1. Ensure the operator PC can reach the GENE board and PTZ controller on the configured IP.
  2. Confirm the PTZ platform is free to move through its full tilt range.
  3. Verify SSH credentials and Pelco-D port are correct in the toolbar.
  4. Set the desired Cycles: count.

4.2 Starting the Test

Click Start. The application immediately:

  • Resets all device tile counters and clears the event log
  • Opens an SSH connection to the GENE board and launches four ping processes (one per Moxa device at 200 ms intervals)
  • Opens a TCP connection to the PTZ controller on the Pelco port
  • Begins cycling the tilt between −90° and +90°

4.3 Cycle Behaviour

Each cycle consists of:

  1. Send absolute position command to −90° → wait 5 seconds
  2. 500 ms dwell
  3. Send absolute position command to +90° → wait 5 seconds
  4. 500 ms dwell
  5. Log Cycle Complete and increment the cycle counter
Note: The 5-second wait per move gives the platform sufficient time to complete the full 180° sweep (~3.5 seconds). Position Reached is logged after each wait regardless of position feedback, as the PTZ controller does not respond to position queries during movement execution.

4.4 Automatic Stop Conditions

The test stops automatically if any of the following occur:

  • The configured cycle count is reached
  • All four monitored devices (10.10.10.2 – 10.10.10.5) have each experienced at least one Connectivity Loss event during the current test run

The user may also click Stop at any time to halt immediately. In all cases, all connections (SSH and Pelco-D TCP) are closed cleanly on termination.

6. Exporting Log Data

Click Export at any time during or after the test. A save dialog will appear. Choose the output format:

  • CSV (.csv) — plain comma-separated file, compatible with any spreadsheet application
  • Excel (.xlsx) — formatted spreadsheet with column headers
Note: The log is written continuously to a temporary file throughout the test. If the application closes unexpectedly, the temporary file is preserved and can be recovered manually from the system temp directory.
